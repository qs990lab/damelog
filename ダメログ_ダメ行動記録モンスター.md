## ダメダメコレクションクエスト（略称：ダメコレ）

「ダメ行動を記録したらモンスターが生まれた。育てていたら、なぜか自分がわかった」

ダメ行動を記録するたびに、AIがその行動を**モンスターとして生成**する。モンスターはダメを重ねるほど変身・進化し、新たな攻撃を覚える。コレクションしたくなって、もっとダメなことをしてしまう——それ自体が「人をダメにする」仕組み。

## モンスター変化システム

### 基本構造: 孵化 → 成長 → 変身 → 覚醒

ダメ行動を入力するたびにモンスターが変化する。変化の方向は「ダメの種類」と「ダメの深刻度」で決まる。

### ダメの種類（属性）

| 属性 | ダメ行動の例 |
|------|------------|
| 💤 怠惰 | 寝坊、二度寝、ゴロゴロ、何もしない |
| 🍕 暴食 | 深夜ラーメン、ポテチ1袋、3食カップ麺 |
| 📱 浪費 | 課金、衝動買い、ガチャ爆死 |
| 🎮 逃避 | Netflix一気見、ゲーム徹夜、SNS3時間 |
| 🍺 酒乱 | 飲みすぎ、記憶喪失、二日酔い |
| 📅 先延ばし | 締切無視、後でやる、明日から本気出す |

### 変身（進化）システム

同じ属性のダメ行動を重ねると、モンスターが段階的に変身する。

```
【怠惰属性の例】

Lv.1 🥚 グータランのタマゴ
　「寝坊した」→ 孵化！

Lv.5 🐣 スヌーズひよこ
　必殺技：「あと5分」

Lv.15 🐲 ネオキドラゴン（変身！）
　必殺技：「二度寝バリア」「目覚まし無効化」
　※寝坊を5回記録で変身

Lv.30 🐉 グランド・フトンリュウ（最終変身！）
　必殺技：「永眠モード」「布団結界」「全アラーム破壊」
　※寝坊を15回 or 「午後まで寝た」を記録で覚醒
```

### 攻撃（必殺技）解放の仕組み

モンスターは特定条件で新しい必殺技を覚える。

| 解放条件 | 覚える技の例 |
|---------|------------|
| 同じダメ行動を3回繰り返す | 基本技が強化（「あと5分」→「あと1時間」） |
| ダメの深刻度が高い行動を記録 | レア技解放（「会議すっぽかし」→「社会的消滅」） |
| 複数属性のダメを同日に記録 | 合体技解放（怠惰+暴食→「布団でピザ無限ループ」） |
| 連続日数記録（3日/5日/7日） | 覚醒技解放（「7日連続ダメ」→「伝説の一撃：人間やめました」） |
| 特定の組み合わせ（隠し条件） | シークレット技（「金曜深夜に課金+飲酒+夜更かし」→「週末崩壊キャノン」） |

### 変身バリエーション案

**案A: 分岐進化型**
同じ属性でも、ダメ行動の内容によって進化先が分岐する。

```
怠惰タマゴ
├→ 寝坊系 → スヌーズドラゴン → グランド・フトンリュウ
├→ 無気力系 → ゴロゴロスライム → 虚無キング
└→ サボり系 → エスケープゴースト → 透明人間マスター
```

**案B: 合体進化型**
異なる属性のモンスター同士が合体して新種が生まれる。

```
スヌーズドラゴン(怠惰) × 深夜ラーメン獣(暴食)
→ 「布団ラーメンキメラ」誕生！
　 必殺技：「寝ながら食う」「カロリー無視フィールド」
```

**案C: 闇落ち進化型**
ダメ度が一定を超えると、モンスターが「闇落ち」して見た目が激変する。

```
Lv.15 かわいいスヌーズひよこ
→ ダメ度MAX突破
→ Lv.15 [DARK] 堕天スヌーズ・ディアボロス
　 必殺技：「永遠の月曜日」「社会復帰不能」
```

## バトルシステム（AI 対戦）

※ バトルシステムは決勝フェーズで実装予定。MVP（予選）ではダメ行動記録→モンスター生成・進化までをスコープとする。

育てたモンスターで AI（NPC）モンスターと非同期バトルができる。

### 仕組み

- AI が生成した NPC モンスターが対戦相手として常時用意されている
- ユーザーは手持ちモンスターを選んでバトルを挑む
- 勝敗はモンスターの Lv・属性相性・必殺技で自動計算
- 勝利すると報酬（図鑑ヒント・レア進化素材など）

### 人をダメにするポイント

- 「もう1戦だけ…」の中毒性
- 勝つためにもっとダメ行動を記録してモンスターを強化したくなる

---

```
1. ユーザーがダメ行動を入力
   「締切明日なのにゲーム8時間やった」

2. AIが分析
   → 属性：逃避 / 深刻度：★★★★ / 追加属性：先延ばし

3. モンスター変化
   → 「ゲームゴブリン」がLv.12→Lv.18に成長！
   → 新技「あと1ステージだけ」を習得！
   → ⚠️ あと2回で変身可能！

4. 図鑑更新
   → 逃避属性コンプリート率: 45%
   → 「隠しモンスターのヒント：金曜の夜に…」

5. ソーシャル通知
   → 「フレンドのAさんが★★★★★モンスターを生み出しました！」
```

## 人をダメにするループ構造

```
ダメ行動する → モンスター変化が楽しい → もっとダメしたくなる
     ↑                                          ↓
     ←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←
```

- **コレクション欲**: 図鑑を埋めたくて、わざとダメなことをする
- **進化欲**: 変身を見たくて、同じダメを繰り返す
- **競争欲**: フレンドより強いモンスターを育てたくて、よりダメになる
- **発見欲**: 隠し進化・シークレット技を見つけたくて、新しいダメに挑戦する

---

## 逆説的価値（審査向けストーリー）

ダメを「記録する」「可視化する」「楽しむ」ことで、結果的に：
- 自分の行動パターンが丸見えになる（自己認知）
- 「またこのダメやってるな」と気づく（メタ認知）
- 図鑑が偏っていることに気づく（弱点の可視化）
- **最終的に「人をダメにする」サービスが、人を少しだけマシにしてしまう**

## 技術構成

### AWS

**Bedrock AgentCore**
- **Runtime**: エージェントのホスティング・スケーリング
- **Memory**: ユーザーのダメ行動パターンや好みをセッション跨ぎで記憶（パーソナライズ）
- **Gateway**: 進化判定・図鑑検索・画像生成呼び出し等のツールを MCP ツールとして一元管理

**Bedrock (LLM)**
- **Amazon Nova 2 Lite / Claude Sonnet 4.6**: ダメ行動テキスト→モンスター属性・名前・技の生成（単純な分類は Nova 2 Lite、複雑な指示や日本語精度が必要な生成は Claude Sonnet 4.6）
- **Guardrails**: ユーザー入力の不適切コンテンツ（自傷・犯罪等）フィルタリング

**認証**
- **Cognito**: ユーザー認証・サインアップ/サインイン

**コンピュート/アプリケーション**
- **Lambda**: 進化判定ロジック・技解放判定
- **EventBridge**: 連続記録日数の判定・通知トリガー

**ストレージ/データ**
- **DynamoDB**: モンスター図鑑・ユーザーデータ・進化ツリー
- **S3**: 生成されたモンスター画像の保存

### AWS 外

**画像生成**
- **Vertex AI - Gemini 3.1 Flash Image (Nano Banana 2)**: モンスターのビジュアル生成

**アプリケーション/フレームワーク**
- **Quart + Jinja2**: Web サーバー・フロントエンド
- **Strands Agents**: エージェント構築フレームワーク
