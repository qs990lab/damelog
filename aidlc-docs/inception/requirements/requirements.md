# 要件定義書: ダメダメコレクションクエスト（ダメコレ）

## Intent Analysis

- **ユーザーリクエスト**: 「人をダメにするサービス」をテーマとしたハッカソン向けアプリケーション開発
- **リクエストタイプ**: 新規プロジェクト
- **スコープ**: 複数コンポーネント（フロントエンド、AIエージェント、データストア、画像生成）
- **複雑度**: Moderate（AI連携・画像生成を含むが、ハッカソンMVPとしてスコープを絞る）

## ビジネス意図（Intent）

「ダメ行動」をネガティブな自己嫌悪ではなく、モンスター育成というゲーミフィケーションに変換することで、記録すること自体を楽しみに変える。ユーザーは「もっとダメしたい」という欲求に駆動されながらサービスを使い続け、結果として自分の行動パターンを無自覚に可視化する。

### 人をダメにするループ構造
- **コレクション欲**: 図鑑を埋めたくて、わざとダメなことをする
- **進化欲**: 変身を見たくて、同じダメを繰り返す
- **発見欲**: 隠し進化・シークレット技を見つけたくて、新しいダメに挑戦する
- **常時アクセス性**: 仕事してる風を装えるビジネスモードにより「仕事中だから使えない」という歯止めが消える。いつでもどこでもダメを記録でき、ループが途切れない

### 逆説的価値
ダメを「記録する」「可視化する」「楽しむ」ことで、結果的に自己認知・メタ認知が促進される。

## 機能要件

### FR-1: ダメ行動記録（チャット形式）
- ユーザーはAIとのチャット形式でダメ行動を記録する
- 初回はフリーテキスト入力でユーザー独自の発想を引き出す
- 過去の入力履歴から選択肢ボタンを生成し、2回目以降はタップで素早く記録可能
- AIが深掘り質問（「何時間？」「昨日もやった？」）で属性・深刻度を精緻化

### FR-2: AI分析・属性判定
- ダメ行動テキストをAIが分析し、属性（怠惰/暴食/浪費/逃避/酒乱/先延ばし）を判定
- 深刻度（★1〜★5）を算定
- 複数属性の同時判定に対応

### FR-3: モンスター生成（グータラン）
- ダメ行動の属性に基づきモンスター（グータラン）を生成
- モンスターには名前・属性・レベル・必殺技が付与される
- AIがモンスターの名前・技名をテーマに沿って生成

### FR-4: モンスター画像
- モンスター画像は事前に多種類を生成しておく（リアルタイム生成ではない）
- キャラクター本体と枠（フレーム）を分離管理
- 属性・進化段階・レア度の組み合わせに応じた画像バリエーションを用意
- 孵化・進化時に対応する画像を表示（演出付き）

### FR-5: 進化・変身システム
- 同じ属性のダメ行動を重ねるとモンスターが段階的に変身
- 分岐進化: ダメ行動の内容によって進化先が分岐
- 合体進化: 異なる属性のモンスター同士が合体して新種誕生
- 闇落ち進化: ダメ度が一定を超えると見た目が激変

### FR-6: 必殺技解放
- 同じダメ行動3回繰り返し → 基本技強化
- 深刻度が高い行動 → レア技解放
- 複数属性同日記録 → 合体技解放
- 連続日数記録（3日/7日/30日） → 覚醒技解放
- 特定の組み合わせ → シークレット技

### FR-7: 図鑑
- 全モンスターの図鑑表示
- 属性別コンプリート率
- 隠しモンスターのヒント表示
- 進化ツリーの可視化

### FR-8: ビジネスモード（偽装UI）
- ぱっと見では仕事をしているように見えるUIモード
- 画面ごとに適切なビジネス風UIを適用:
  - チャット画面 → Slack/Teams風（業務チャットに見える対話画面）
  - 図鑑・モンスター一覧 → Excel/スプレッドシート風（データテーブル表示）
  - ステータス・進捗 → ダッシュボード/BI風（グラフ・KPIカード表示）
- ワンタップで通常UI/ビジネスUIを切り替え

### FR-9: モンスター画像構成（キャラクター + 枠）
- モンスター画像はキャラクター本体と枠（フレーム）を分離して管理
- 枠はレア度に応じたデザイン（ノーマル/レア/スーパーレア/ウルトラレア等）
- キャラクター画像はAI生成、枠は事前デザインのテンプレート
- 組み合わせて表示することでカード風の見た目を実現

### FR-10: レア度システム
- モンスターにレア度を付与（N/R/SR/UR の4段階）
- 高度なステータス（高Lv進化、シークレット技、合体進化等）はレア度が高い
- レア度に応じて枠のデザインが変化
- UR獲得時は特別な演出（画面エフェクト、枠の光り等）を表示
- 図鑑でレア度別フィルタリング可能
- レア度が高いモンスターほど生成条件が厳しい（特定の組み合わせ・連続日数等）

### FR-10.1: 演出システム
- 孵化時: 卵が揺れる→割れる→モンスター登場の演出
- 進化時: 光る→シルエット変化→新形態お披露目の演出
- UR獲得時: 虹色エフェクト・特別SE等の豪華演出
- 技解放時: 技名表示＋エフェクト演出

### FR-11: ユーザー認証
- Cognito による標準的なサインアップ/サインイン（メール+パスワード）
- セッション管理

### FR-12: バトルシステム（決勝フェーズで実装）
- 育てたモンスターでAI（NPC）モンスターと非同期バトル
- 勝敗はLv・属性相性・必殺技で自動計算
- 勝利報酬（図鑑ヒント・レア進化素材）

## 非機能要件

### NFR-1: パフォーマンス
- ダメ行動入力→モンスター属性判定: 3秒以内
- 画像生成: 10秒以内（非同期処理可）
- 図鑑表示: 2秒以内

### NFR-2: 可用性
- ハッカソンデモとして安定動作すること
- 同時接続: 10ユーザー程度（デモ用）

### NFR-3: データ永続性
- ユーザーのダメ行動履歴は無期限保持
- モンスターデータ・図鑑データは永続化

### NFR-4: コンテンツ安全性
- Bedrock Guardrails によるユーザー入力フィルタリング
- 自傷・犯罪等の不適切コンテンツをブロック

### NFR-5: 言語
- 日本語のみ対応

## 技術構成

### AWS
| サービス | 用途 |
|---------|------|
| Bedrock AgentCore Runtime | エージェントのホスティング・スケーリング |
| Bedrock AgentCore Memory | ユーザーのダメ行動パターン記憶（パーソナライズ） |
| Bedrock AgentCore Gateway | ツール（進化判定・図鑑検索・画像生成呼び出し等）をMCPツールとして一元管理 |
| Bedrock (Nova 2 Lite / Claude Sonnet 4.6) | ダメ行動分析・モンスター名/技名生成 |
| Bedrock Guardrails | 不適切コンテンツフィルタリング |
| Cognito | ユーザー認証 |
| Lambda | Gateway経由で呼び出すツール群（進化判定、図鑑検索等） |
| EventBridge | 連続記録日数判定・通知トリガー |
| DynamoDB | モンスター図鑑・ユーザーデータ・進化ツリー |
| S3 | 生成されたモンスター画像の保存 |

### AWS外
| サービス | 用途 |
|---------|------|
| Vertex AI - Gemini 3.1 Flash Image | モンスターのビジュアル生成 |

### フレームワーク
| 技術 | 用途 |
|------|------|
| Quart + Jinja2 | Webサーバー・フロントエンド |
| Strands Agents | エージェント構築フレームワーク |
